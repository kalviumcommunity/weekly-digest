# 🤝 Contributing to Weekly Tech Digest Scraper

First off, thank you for considering contributing to this project! It's people like you that make the tech community such a vibrant place. 🚀

## 🌈 How Can I Contribute?

### 🐛 Reporting Bugs
- Check the [GitHub Issues](https://github.com/kalviumcommunity/weekly-digest/issues) to see if the bug has already been reported.
- If not, create a new issue. Provide as much detail as possible, including steps to reproduce the bug and your environment details.

### ✨ Suggesting Enhancements
- Open an issue with the tag `enhancement`.
- Describe the feature you'd like to see and why it would be useful.

### 📝 Improving Documentation
- Documentation is just as important as code! If you find a typo or think a section could be clearer, feel free to open a Pull Request.

## 🛠️ Development Setup

1. **Fork the repository** to your own GitHub account.
2. **Clone the fork** to your local machine:
   ```bash
   git clone https://github.com/YOUR_USERNAME/weekly-digest.git
   ```
3. **Install dependencies**:
   ```bash
   pip install -r requirements.txt
   ```
4. **Create a new branch** for your changes:
   ```bash
   git checkout -b feature/amazing-new-feature
   ```

## 🧪 Testing Your Changes

Before submitting your PR, please ensure your changes work as expected:
- Run `python report_generator.py` to verify the scraper and HTML generation.
- Check that `output/weekly_digest.html` looks correct.

## 🚀 Submitting a Pull Request

1. **Commit your changes** with clear, descriptive commit messages.
2. **Push your branch** to your fork:
   ```bash
   git push origin feature/amazing-new-feature
   ```
3. **Open a Pull Request** from your fork's branch to our `main` branch.
4. **Wait for review!** We'll try to review your PR as soon as possible.

## 📜 Code of Conduct
By participating in this project, you agree to abide by our code of conduct. Please be respectful and inclusive of all contributors.

Happy Coding! 💻✨
